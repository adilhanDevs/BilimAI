# Документация моделей приложения `lessons`

В данном документе описаны все модели, используемые в приложении `lessons`, за исключением моделей шагов уроков (`LessonStep` и связанные с ним специализированные таблицы). База данных разделена на логические блоки: структура курса, локализация, медиа-ресурсы контента и отслеживание прогресса пользователей.

## 1. Структура курса (`models/course.py`)

*   **`Course` (Курс)**: Основная сущность учебного курса (например, курс кыргызского языка для англоговорящих). Содержит метаданные курса, целевой и исходный языки общения, а также диапазоны уровней CEFR.
*   **`Unit` (Модуль/Раздел)**: Крупный логический блок внутри курса. Может объединять несколько уроков или тем по порядку.
*   **`Category` (Категория/Тема)**: Группировка схожих уроков (в Дуолинго это "кружочки" на карте, например "семья", "еда"). Поддерживают иерархию через `prerequisite_category` для открытия новых тем по мере прохождения, а также указывают сложность и объем знаний (ожидаемое время, число новых слов).
*   **`Lesson` (Урок)**: Основная единица обучения. Привязывается к категории (или к `Unit`). Включает информацию об уровне сложности, наградах за прохождение (базовый XP) и грамматическом фокусе.
*   **`LessonVocabulary` (Словарь урока)**: Модель для хранения списка слов и выражений, которые изучаются или встречаются в рамках конкретного урока (включает само слово, перевод, части речи, примеры и аудио-произношение).

## 2. Локализация и переводы (`models/localization.py`)

*   **`Language` (Язык)**: Справочник поддерживаемых языков в формате стандартных ISO-кодов ("en", "ky", "ru") и их читаемых названий.
*   **`TranslationGroup` (Группа переводов)**: Объединяющая сущность для концептов или интерфейсных фраз, требующих перевода. Может содержать контекстные комментарии (context notes) для переводчиков.
*   **`Translation` (Перевод)**: Конкретный текстовый перевод абстрактно выделенного текста (из `TranslationGroup`) на один из доступных языков из `Language`.

## 3. Базовый контент и медиа-сопровождение (`models/engine.py`)

*   **`Asset` (Медиафайл)**: Централизованная модель для хранения медиаданных (аудио, изображения, видео), используемых движком уроков. Содержит файл, тип контента, продолжительность (для аудио/видео) и JSON поле для дополнительных метаданных.
*   **`ContentUnit` (Единица контента)**: Фундаментальный "атом" обучающего материала (слово, фраза или целое предложение). Составной элемент: включает в себя сам текст, перевод (с помощью fallback значений или связей с `TranslationGroup`), а также ссылки на медиа-информацию: главную аудиозапись (`primary_audio`) и картинку-ассоциацию (`primary_image`).

## 4. Сессии, Игрофикация и Прогресс (`models/progress.py`)

*   **`LessonSession` (Сессия урока)**: Запись об активной, успешно завершенной, проваленной или прерванной попытке прохождения урока конкретным пользователем. Отвечает за состояние "игры": оставшиеся жизни (hearts), количество совершенных ошибок, заработанный XP в процессе и количество пройденных шагов.
*   **`StepAttempt` (Попытка выполнения шага)**: Логирование каждого ответа (или действия) пользователя на конкретный шаг `LessonStep` в ходе активной сессии. Содержит статус правильности, баллы и исходный payload, отправленный клиентом.
*   **`SpeechSubmission` (Распознавание речи)**: Специфичная модель для хранения записей устной речи пользователя и процесса её оценки асинхронным провайдером. Имеет статусы обработки (например `pending`, `processing`) и сохраняет сырой ответ от движка оценки.
*   **`UserLessonProgress` (Прогресс прохождения урока)**: Сводная статистика пользователя по определенному уроку в целом (статус завершения, лучший результат, суммарно заработанный XP за все попытки, дата последней активности).
*   **`UserContentProgress` (Прогресс заучивания слов/фраз)**: Отвечает за систему интервального повторения на уровне конкретных `ContentUnit`. Хранит информацию об уровне усвоения, частоте столкновений (вплоть до числа правильных/ложных ответов) и дате следующего необходимого повторения слова.
*   **`CourseEnrollment` (Запись на курс)**: Отражает факт записи пользователя на изучение курса.
*   **`UserCategoryProgress` (Прогресс по категориям)**: Отражает статус открытия определенной темы у пользователя (тема заблокирована, доступна, в процессе изучения или полностью освоена) и % завершения.
*   **`ReviewItem` (Элемент на детальное повторение)**: Модель для формирования очереди повторений пользователя (сохранение наболевших/забытых слов). Хранит карточки для повторения, дату следующей тренировки (due date), количество прошлых ошибок и текущую "силу" знания.
*   **`UserSkillProgress` (Прогресс по навыкам)**: Раздельная статистика развития по ключевым навыковым областям пользователя (Чтение, Письмо, Аудирование, Говорение, Словарный запас, Грамматика) в рамках конкретного курса — очки и текущий уровень навыка.
