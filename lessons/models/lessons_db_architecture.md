# Архитектура Базы Данных: Приложение `lessons`

В данном документе подробно описана структура базы данных для обучающего приложения (модуль `lessons`). Модели логически разделены на четыре основные подсистемы: структура курсов, мультиязычность (локализация), движок упражнений (шаги уроков) и отслеживание пользовательского прогресса.

---

## 1. Иерархия учебного контента (`course.py`)
Определяет базовую структуру обучения от макроуровня (Курс) до микроуровня (Урок и его словарь).

### `Course` (Курс)
Верхнеуровневая сущность, представляющая языковой курс (например, "Турецкий для кыргызов").
* **Поля:**
  * `id` *(UUID)*: Первичный ключ.
  * `slug` *(String, Уникальное)*: ЧПУ (постоянная ссылка).
  * `source_language` *(String)*: Язык инструкций/преподавания (например, `ky` или `en`).
  * `target_language` *(String)*: Изучаемый язык (например, `tr`).
  * `title` *(String)*: Название курса.
  * `description_ky` *(Text)*: Описание курса на кыргызском языке.
  * `cefr_min` / `cefr_max` *(String)*: Поддерживаемый диапазон уровней языка (например, `A1` — `B2`).
  * `is_published` *(Boolean)*: Флаг доступности курса для пользователей.
  * `created_at`, `updated_at` *(DateTime)*: Временные метки.

### `Unit` (Модуль / Раздел курса)
Тематический или уровневый блок внутри курса.
* **Поля:**
  * `id` *(UUID)*: Первичный ключ.
  * `slug` *(String)*: Строка-идентификатор. Уникальна в рамках одного курса.
  * `title` *(String)*: Название модуля.
  * `description_ky` *(Text)*: Описание модуля на кыргызском.
  * `sort_order` *(Integer)*: Порядковый номер для сортировки (выстраивает модули друг за другом).
  * `cefr_level` *(String)*: Уровень CEFR для этого конкретного модуля.
* **Связи:**
  * `course` *(ForeignKey на Course)*: Курс, к которому принадлежит модуль (обратная связь `units`).

### `Category` (Категория / Подраздел)
Группирует уроки внутри курса. Может выстраиваться в цепочку зависимости (нужно пройти предыдущую, чтобы открыть следующую).
* **Поля:**
  * `id` *(UUID)*: Первичный ключ.
  * `slug` *(String, Уникальное)*: ЧПУ категории.
  * `title_ky` *(String)*: Заголовок на кыргызском.
  * `title_target` *(String)*: Заголовок на изучаемом языке.
  * `description_ky` *(Text)*: Описание на кыргызском.
  * `icon` *(Text)*: Название или URL иконки.
  * `difficulty` *(String)*: Уровень сложности (по умолчанию `beginner`).
  * `sort_order` *(Integer)*: Порядок сортировки.
  * `estimated_minutes` *(Integer)*: Ожидаемое время прохождения (в минутах).
  * `vocabulary_count` *(Integer)*: Ожидаемое количество изучаемых слов.
  * `is_premium` *(Boolean)*: Доступна ли категория только по платной подписке.
* **Связи:**
  * `course` *(ForeignKey на Course)*: Курс, к которому относится категория (обратная связь `categories`).
  * `prerequisite_category` *(ForeignKey на Category, Self-referential)*: Категория, которую необходимо пройти до начала текущей.

### `Lesson` (Урок)
Конкретное занятие внутри категории и/или модуля.
* **Поля:**
  * `id` *(UUID)*: Первичный ключ.
  * `slug` *(String, Уникальное)*: ЧПУ урока.
  * `title` *(String)*: Заголовок.
  * `subtitle` *(String)*: Подзаголовок (опционально).
  * `description_ky` *(Text)*: Описание урока на кыргызском.
  * `level` *(String)*: Уровень (например, `a0`).
  * `difficulty` *(String)*: Сложность (`beginner`).
  * `duration_min` *(Integer)*: Примерная продолжительность урока в минутах.
  * `xp_reward` *(Integer)*: Базовое количество опыта (XP), которое дается за прохождение.
  * `grammar_focus` *(Text)*: Описание грамматической темы (если есть).
  * `is_premium` *(Boolean)*: Платный ли урок.
  * `is_published` *(Boolean)*: Доступен ли урок в приложении.
  * `sort_order` *(Integer)*: Порядок прохождения урока.
* **Связи:**
  * `category` *(ForeignKey на Category)*: Родительская категория (обратная связь `lessons`).
  * `unit` *(ForeignKey на Unit, Опционально)*: Родительский модуль (обратная связь `lessons`).

### `LessonVocabulary` (Словарь урока)
Слова и фразы, которые изучаются в рамках конкретного урока.
* **Поля:**
  * `id` *(UUID)*: Первичный ключ.
  * `word_or_phrase_target` *(String)*: Слово или фраза на изучаемом языке.
  * `translation_ky` *(String)*: Перевод на кыргызский язык.
  * `transliteration` *(String)*: Транслитерация (произношение, опционально).
  * `part_of_speech` *(String)*: Часть речи.
  * `example_target` *(Text)*: Пример использования на изучаемом языке.
  * `example_ky` *(Text)*: Перевод примера на кыргызский.
  * `audio_url` *(URL)*: Ссылка на озвучку слова.
  * `difficulty` *(String)*: Сложность.
  * `sort_order` *(Integer)*: Порядок в списке слов урока.
* **Связи:**
  * `lesson` *(ForeignKey на Lesson)*: Урок, в котором изучается слово.
  * `content_unit` *(ForeignKey на ContentUnit, Опционально)*: Связь с глобальной словарной базой движка.

## 2. Локализация и Переводы (`localization.py`)
Обеспечивает мультиязычность контента внутри упражнений (например, инструкций разным пользователям на их родных языках).

### `Language` (Язык)
* **Поля:** `code` (String, PK, например `en`, `ky`, `ru`), `name` (String, Название).

### `TranslationGroup` (Группа переводов)
Абстрактная база для связи переводов одной и той же сущности на разные языки. Заменяет прямое указание текста в поле.
* **Поля:** `id` (UUID), `context_note` (Контекст/подсказка для переводчиков).

### `Translation` (Перевод)
Фактический текстовый перевод для определенной группы и языка.
* **Поля:** `text` (Text). Находится в уникальной связке `(group, language)`.
* **Связи:** `group` (ForeignKey на TranslationGroup), `language` (ForeignKey на Language).

## 3. Движок уроков и упражнений (`engine.py`)
Система, собирающая интерактивные задания внутри урока.

### `Asset` (Медиафайл)
Файлы аудио, видео и изображений.
* **Поля:** `id`, `asset_type` (`audio`, `image`, `video`), `file` (Путь к файлу), `duration_ms` (Длительность), `metadata` (JSON).

### `ContentUnit` (Единица контента)
Универсальный строительный блок для фраз и слов, который используется в заданиях (позволяет переиспользовать контент).
* **Поля:** `id`, `unit_type` (`word`, `phrase`, `sentence`), `text` (Запасной текст, если нет перевода), `meaning` (Запасное значение/подсказка).
* **Связи:**
  * `text_group` / `meaning_group` *(ForeignKey на TranslationGroup)*: Локализованные версии текста и значения.
  * `primary_audio` / `primary_image` *(ForeignKey на Asset)*: Основные изображения и аудио для этого слова/фразы.

### `LessonStep` (Шаг урока)
Один экран (задание) в процессе прохождения урока.
* **Поля:**
  * `id` *(UUID)*
  * `step_type` *(String)*: Тип шага (например, `multiple_choice`, `fill_blank`, `speak_phrase`). Валидируется через `StepRegistry`.
  * `prompt`, `instruction` *(Text)*: Текстовые вопросы/инструкции.
  * `xp_reward` *(Integer)*: Ревард за этот шаг (по умолчанию 10).
  * `sort_order` *(Integer)*: Очередность шага в уроке.
* **Связи:**
  * `lesson` *(ForeignKey на Lesson)*: Какому уроку принадлежит.
  * `prompt_group`, `instruction_group` *(ForeignKey на TranslationGroup)*: Локализованные версии инструкций.

### Детали шагов урока
Объекты, которые связаны с `LessonStep` связью **OneToOne** (в зависимости от `step_type`) и хранят данные, специфичные для этого типа задания.

* **1. `StepMultipleChoice` (Тест с вариантами ответа)**
  * Содержит множество **`StepChoice`** — вариантов ответа.
  * *StepChoice*: содержит `text` или ссылку на `ContentUnit`, а также флаг `is_correct` (правильный ли ответ) и порядковый номер `sort_order`.

* **2. `StepFillBlank` (Заполни пропуск)**
  * *Поля:* `source_unit` (привязанный ContentUnit), `sentence_template` (шаблон предложения с токеном `[[blank]]`), `acceptable_answers` (JSON-массив списка правильных строковых вариантов текста для пропуска).

* **3. `StepMatchPairs` (Сопоставь пары)**
  * Содержит множество **`MatchPairItem`** — пар для соединения (например, с одной стороны слово на кыргызском, с другой на турецком).
  * *MatchPairItem*: левая половина (`left_content_unit` / `left_text`), правая половина (`right_content_unit` / `right_text`), `sort_order`.

* **4. `StepReorderSentence` (Составь предложение из частей)**
  * Содержит множество **`ReorderToken`** — кубики/слова из которых надо собрать текст.
  * *ReorderToken*: `content_unit` или `text`, флаг `is_distractor` (явлется ли слово ложью/лишним, которое не нужно использовать), `sort_order` (правильный порядок).

* **5. `StepTypeTranslation` (Напиши перевод текстом)**
  * *Поля:* `source_unit` / `source_text` (то, что надо перевести), `acceptable_answers` (JSON-массив всех допустимых, правильных строковых вариантов перевода).

* **6. `StepSpeakPhrase` (Произнеси фразу — проверка речи)**
  * *Поля:* `target_unit` / `target_text` / `target_text_group` (фраза, которую нужно произнести), `reference_audio` (аудио-пример, как это должно звучать).
  * *Настройки шага:* `min_score_required` (минимальный порог распознавания в процентах для успеха, дефолт 70), `allow_retry` (можно ли пробовать еще).

## 4. Прогресс пользователей (`progress.py`)
Хранение всех взаимодействий пользователя с курсами, уроками и заданиями для аналитики и геймификации.

### Детальные попытки (Сессии)
* **`LessonSession` (Игровая сессия урока)**: Создается, когда пользователь запускает урок.
  * *Поля:* `status` (`active`, `completed`, `failed`, `abandoned`), `hearts_remaining` (оставшиеся жизни для текущей сессии), `mistakes_count`, `xp_earned`, `completed_steps_count`, `started_at`, `completed_at`.
  * *Связи:* `user` (Пользователь), `lesson` (Урок).
* **`StepAttempt` (Попытка ответа на шаг)**:
  * *Поля:* `is_correct` (успешно ли), `score` (баллы), `client_payload` (JSON того, что прислал клиент — введенный текст, выбранный вариант).
  * *Связи:* `session` (Сессия), `step` (Конкретный LessonStep).
* **`SpeechSubmission` (Попытка голосового ввода)**: Расширенный лог для заданий с произношением.
  * *Поля:* `status` (от `pending` до `completed`), `audio_file` (запись речи), `provider` (движок распознавания), `external_job_id`, `final_score` (оценка звука), `raw_result` / `feedback_payload` (ответ от AI для подробного разбора ошибок пользователю).

### Состояние прогресса сущностей (Сохранения)
* **`CourseEnrollment`**: Запись студента на курс (статус `is_active`, даты).
* **`UserCategoryProgress`**: Общий прогресс по категории.
  * *Поля:* `status` (`locked`, `available`, `in_progress`, `completed`), `progress_percent`, `completed_lessons`, `total_lessons`.
* **`UserLessonProgress`**: Глобальный статус урока для конкретного пользователя.
  * *Поля:* `status` (`not_started`, `in_progress`, `completed`, `mastered`), `best_score`, `total_sessions`, `total_attempts`, `total_correct_attempts`, `total_xp_earned`.
* **`UserContentProgress`**: История изучения конкретного слова/фразы (`ContentUnit`).
  * *Поля:* `exposure_count` (сколько раз видел), `correct_count`, `incorrect_count`, `mastery_score`, `next_review_at` (для алгоритмов интервального повторения).
* **`UserSkillProgress`**: Очки навыка пользователя (`reading`, `writing`, `listening`, `speaking`, `vocabulary`, `grammar`) внутри определенного курса.
  * *Поля:* `score` (текущие очки), `level` (уровень навыка).
* **`ReviewItem` (Элемент для повторения)**: Элемент алгоритма Spaced Repetition (интервальное повторение).
  * *Поля:* `item_type`, `target_text`, `translation_ky`, `due_at` (когда нужно повторить снова), `strength` (сила памяти), `correct_streak`, `is_completed`.
  * *Связи:* `content_unit`, `lesson`, `vocabulary` (что конкретно надо вспомнить).
